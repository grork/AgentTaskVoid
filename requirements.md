Things that we must have that are otherwise uncaptured:
- A 'clear' command that removes all AppTaskInfos that have been created by this application, and it should clear any internal state of the application that is tracking these across invocations
- Expiry/cleanup passes decide on FRESH state, AFTER active work. Within any cycle (watchdog poll or CLI invocation), apply pending writes/reconciliation first, THEN run the expiry pass, which re-reads each handle's `lastUpdate` (under the INFRA-6 write mutex) immediately before comparing to now -- never from a snapshot taken earlier in the cycle. Compute task age from the wall-clock `lastUpdate` timestamp (which correctly includes time the device spent asleep), not a monotonic/paused timer or a pre-sleep cached "expires-at" deadline; on wake, recompute and trust no pre-sleep timer. This closes the sleep/wake (and general interleave) window where a just-updated task is reaped on stale `lastUpdate` -- LIFE-21 recycle-bin resurrection makes a slip graceful, but the ordering makes it rare. Cross-ref LIFE-16, LIFE-21, INFRA-6.
- `clear` purges all active handles (every task + its sidecar entry + icon copy) but NOT the LIFE-21 recycle bin; a `--include-recycle-bin` flag extends the purge to the recycle bin. LIFE-20's boot-recovery clear is the exception -- it wipes the recycle bin per LIFE-21. (Ratified 2026-07-05; recorded in ERGO-16.)
