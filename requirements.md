Requirements not captured elsewhere:
- A `clear` command that removes every AppTaskInfo this application created, and clears the application's own internal state tracking them across invocations.
- Expiry/cleanup passes decide on fresh state, after active work. Within any cycle (watchdog poll or CLI invocation), apply pending writes/reconciliation first, then run the expiry pass, which re-reads each handle's `lastUpdate` (under the INFRA-6 write mutex) immediately before comparing to now -- never from a snapshot taken earlier in the cycle. Compute task age from the wall-clock `lastUpdate` timestamp (which includes time the device spent asleep), not a monotonic/paused timer or a pre-sleep cached "expires-at" deadline; on wake, recompute and trust no pre-sleep timer. This closes the sleep/wake (and general interleave) window where a just-updated task is reaped on stale `lastUpdate` -- LIFE-21 recycle-bin resurrection makes a slip graceful, but the ordering makes it rare. Cross-ref LIFE-16, LIFE-21, INFRA-6.
- v2 fan-out glomming needs a shared icon URI across handles. Grouping is keyed on the exact `IconUri` string (ERGO-13), but `IconService.Place` writes a per-handle icon path (ERGO-15 separate-by-handle default), so two handles never share a URI and never glom. A subagent child card (handle `<session>#<agent_id>`) must therefore reuse the parent session's exact `IconUri` and pass it straight to `Create` -- never call `IconService.Place` for a child, which would mint a distinct per-child path and break glomming (ERGO-31 §4 fan-out depends on this). Cross-ref ERGO-13/15/22, ERGO-31 §4, LIFE-24.
- `clear` purges all active handles (every task + its sidecar entry + icon copy) but not the LIFE-21 recycle bin; a `--include-recycle-bin` flag extends the purge to the recycle bin. LIFE-20's boot-recovery clear is the exception -- it wipes the recycle bin per LIFE-21. Recorded in ERGO-16.
